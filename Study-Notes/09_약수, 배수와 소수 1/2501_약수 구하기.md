# 2026-01-05
## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **요약**: 자연수 $N$과 $K$가 주어졌을 때, $N$의 약수들 중에서 $K$번째로 작은 수를 출력하는 문제.
- **핵심 조건**: 
	- $N$을 나누어 떨어지게 하는 수($N \% i == 0$)가 약수이다.
	- 만약 $N$의 약수의 개수가 $K$개보다 적어서 $K$번째 약수가 존재하지 않을 경우 `0`을 출력해야 한다.

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1. **변수 설정:** 
	- 입력받을 수 `a`($N$), `b`($K$).
	- 찾은 약수의 순서를 셀 `count`.
	- 결과값을 저장할 `result` (초기값은 0으로 설정하여 약수가 부족할 때 자연스럽게 0이 출력되도록 함).

2. **로직 설계:** 
	- 1부터 `a`까지 반복문(`for`)을 돌린다.
	- `a`를 `i`로 나눴을 때 나머지가 0이면(`a % i == 0`) 약수이므로 `count`를 1 증가시킨다.
	- `count`가 찾으려는 순서 `b`와 같아지면, 현재의 `i`가 정답이므로 `result`에 저장하고 반복문을 탈출(`break`)한다.

3. **시간 복잡도:** 
	- 1부터 $N$까지 반복하므로 $O(N)$.
	- $N$이 최대 10,000이므로 시간 내에 충분히 연산 가능하다.

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.util.StringTokenizer;

public class Main {

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
        StringTokenizer st;
		
        st = new StringTokenizer(br.readLine());
		
        int a = Integer.parseInt(st.nextToken()); // N
        int b = Integer.parseInt(st.nextToken()); // K
		
        int count = 0, result = 0; // result를 0으로 초기화하여 예외 케이스 처리
		
        for (int i = 1; i <= a; i++) {
            if (a % i == 0) { // 약수일 경우
                count++;
            }
            if (count == b) { // K번째 약수를 찾은 경우
                result = i;
                break; // 더 이상 탐색할 필요 없으므로 종료
            }
        }
        System.out.println(result);
    }
}
```

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **성공 요인:** 
	- `result` 변수를 0으로 초기화해 둔 덕분에, 반복문이 끝날 때까지 $K$번째 약수를 찾지 못해도 별도의 `else` 처리 없이 문제 조건(0 출력)을 만족시킬 수 있었음.
	- `break`문을 사용하여 불필요한 반복을 줄임.
- **배운 점:**
	- `Scanner` 대신 `BufferedReader`와 `StringTokenizer`를 사용하여 입력 속도를 높이는 방식을 적용해 봄.
	- 약수를 구하는 가장 기초적인 방법($O(N)$)을 확실히 복습함.
- **다른 사람 풀이 참고:**
	- $N$이 매우 클 경우($10$억 등)에는 $\sqrt{N}$까지만 반복하여 약수를 구하는 $O(\sqrt{N})$ 알고리즘을 사용해야 하지만, 이 문제는 $N \le 10,000$이라 현재 풀이가 가장 직관적이고 적절함.
