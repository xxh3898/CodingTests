# 2026-01-08
## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **요약**: 자연수 $M$과 $N$이 주어졌을 때, $M$ 이상 $N$ 이하의 자연수 중 소수인 것을 모두 찾아 그 합과 최솟값을 구하는 문제.
- **핵심 조건**: 
  - 소수가 없을 경우에는 `-1`을 출력해야 한다.
  - 소수는 약수가 1과 자기 자신, 딱 2개인 수이다. (1은 소수가 아님)
  - $M \le N \le 10,000$

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1. **변수 설정:**
	- 입력 범위 `n1` ~ `n2`.
	- 소수들의 합을 저장할 `sum` (초기값 0).
	- 최솟값을 저장할 `min`. (초기값은 입력 범위의 최댓값보다 큰 `10001`로 설정하여 첫 소수가 무조건 저장되게 함).

2. **알고리즘 설계 (이중 반복문):**
	- `n1`부터 `n2`까지 숫자를 하나씩 순회한다 (`i`).
	- 각 숫자 `i`에 대해 1부터 `i`까지 나누어 약수의 개수를 센다 (`count`).
	- `count == 2`라면 소수이므로:
	    - `sum`에 `i`를 더한다.
	    - `i`가 현재 `min`보다 작다면 `min`을 갱신한다. (범위가 오름차순이므로 사실상 첫 번째 발견된 소수가 최솟값이 됨).
	- 반복 종료 후 `sum`이 0이면 소수가 없다는 뜻이므로 `-1` 출력, 아니면 `sum`과 `min` 출력.

3. **시간 복잡도:** 최악의 경우 $O(N^2)$에 가깝지만, $N$이 최대 10,000이라 시간 내 통과 가능.

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;

public class Main {

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));

        int n1 = Integer.parseInt(br.readLine()); // M
        int n2 = Integer.parseInt(br.readLine()); // N
        int sum = 0;
        int min = 10001; // N의 최대값이 10000이므로 그보다 큰 값으로 초기화

        // M부터 N까지 순회
        for (int i = n1; i <= n2; i++) {
            int count = 0;

            // 약수 개수 구하기
            for (int j = 1; j <= i; j++) {
                if (i % j == 0) {
                    count++;
                }
            }

            // 약수가 2개면 소수
            if (count == 2) {
                sum += i;

                // 최솟값 갱신 (가장 먼저 발견된 소수가 최솟값이 됨)
                if (i < min) {
                    min = i;
                }
            }
        }

        // 결과 출력
        if (sum == 0) { // 소수가 없는 경우
            bw.write(-1 + "");
        } else {
            bw.write(sum + "\n" + min);
        }

        bw.close();
        br.close();
    }
}
```

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **성공 요인:** 
	- `min` 변수의 초기값을 입력 가능한 최댓값보다 크게 설정하여, 첫 소수 발견 시 무조건 갱신되도록 로직을 잘 짰음.
	- 소수가 없을 때 `-1`을 출력하는 예외 처리를 잊지 않고 구현함.
  
- **배운 점:** 구간 내의 소수를 찾을 때, `count == 2` 조건을 사용하면 1(약수 1개)을 자연스럽게 걸러낼 수 있어 편리하다.

- **다른 사람 풀이 참고 (최적화):**
	- 현재 풀이는 모든 수에 대해 약수를 전부 세므로 비효율적일 수 있음.
	- **제곱근 이용 ($O(N\sqrt{N})$)**: 내부 반복문을 `j * j <= i`까지만 돌리면 훨씬 빨라짐.
	- **에라토스테네스의 체**: 미리 배열을 만들어 소수가 아닌 수(배수)를 지워나가는 방식을 쓰면 가장 빠름.````
