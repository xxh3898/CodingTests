# 2026-01-16
## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **요약**: $N$개의 점(옥구슬)이 주어졌을 때, 이 점들을 모두 감싸는 가장 작은 직사각형의 넓이를 구하는 문제.
- **핵심 조건**: 
	- 직사각형의 변은 좌표축에 평행하다.
	- 넓이 = $(\text{가로 길이}) \times (\text{세로 길이})$
	- 가로 길이 = $(X_{\text{max}} - X_{\text{min}})$
	- 세로 길이 = $(Y_{\text{max}} - Y_{\text{min}})$

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1. **메모리 초과 원인 분석:**
	- 코드에서 `for` 반복문 안에 `int[] arrX = new int[n];` 선언이 들어있음
	- 이렇게 되면 $N$번 반복할 때마다 크기가 $N$인 배열을 계속 새로 생성
	- 만약 $N=100,000$이라면, 불필요하게 거대한 메모리를 계속 잡아먹게 되어 **메모리 초과**가 발생

2. **알고리즘 재설계:**
   - **배열이 필요 없음**: 모든 점의 좌표를 저장할 필요가 없음. 입력받는 즉시 최솟값과 최댓값만 갱신하면 됨
   - **변수 초기화 주의**: 좌표값은 음수($-10,000$)도 가능하므로, 최댓값(`max`)을 찾기 위한 초기 변수는 아주 작은 값으로, 최솟값(`min`)을 찾기 위한 변수는 아주 큰 값으로 설정해야 함.

3. **시간 복잡도:**
   - 점의 개수 $N$만큼 한 번만 순회하므로 $O(N)$.

## 💻 소스 코드 및 문제 설명

```java
import java.io.BufferedReader;
import java.io.BufferedWriter;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.OutputStreamWriter;
import java.util.StringTokenizer;

public class Main {

    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        BufferedWriter bw = new BufferedWriter(new OutputStreamWriter(System.out));
        StringTokenizer st;

        int n = Integer.parseInt(br.readLine());

        // 초기화 주의: 좌표는 -10,000 ~ 10,000 범위임
        int minX = 10001, minY = 10001; 
        int maxX = -10001, maxY = -10001; // -1로 초기화하면 음수 좌표일 때 오류 발생

        for (int i = 0; i < n; i++) {
            st = new StringTokenizer(br.readLine());
            
            // 배열 없이 입력 즉시 비교
            int x = Integer.parseInt(st.nextToken());
            int y = Integer.parseInt(st.nextToken());

            if (minX > x) minX = x;
            if (maxX < x) maxX = x;
            if (minY > y) minY = y;
            if (maxY < y) maxY = y;
        }

        // 넓이 구하기
        bw.write((maxX - minX) * (maxY - minY) + "");

        bw.close();
        br.close();
    }
}
```

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **틀린 이유 (메모리 초과):** 
	- `for`문 내부에서 `new int[n]`을 선언하여, 반복 횟수만큼 배열 객체가 무수히 생성됨. 
	- $N$이 클 경우 힙 메모리를 과도하게 사용하여 터짐.
- **틀린 이유 (논리 오류):**
	- `maxX`를 `-1`로 초기화했는데, 만약 입력된 좌표가 모두 `-5`라면 `maxX`는 여전히 `-1`로 남게 되어 계산이 틀림. (초기값은 가능한 범위 밖의 가장 작은 수로 해야 함).
- **배운 점:** 
	- **입력받은 데이터를 모두 저장할 필요가 있는지** 먼저 고민하자. (최댓값/최솟값만 필요하면 변수 2개면 충분하다).
	- 반복문 안에서의 객체 생성(`new`)은 성능과 메모리에 치명적일 수 있다.
