## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **요약**: 이중 `for`문으로 구성된 알고리즘의 수행 횟수와 시간 복잡도(다항식의 최고차항 차수)를 출력하는 문제.
- **핵심 조건**: 
  - 알고리즘 구조:
    ```
    for i = 1 to n
        for j = 1 to n
            ...
    ```
  - 입력 $n$의 범위: $1 \le n \le 500,000$.
  - 첫째 줄에 수행 횟수를, 둘째 줄에 차수를 출력한다.

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1.  **수행 횟수 계산**:
    - 외부 반복문이 $n$번, 내부 반복문이 $n$번 돈다.
    - 총 수행 횟수 = $n \times n$.
    
2.  **자료형 선택 (매우 중요)**:
    - $n$의 최댓값은 $500,000$.
    - $n^2$은 $250,000,000,000$ (2,500억).
    - `int` 범위(약 21억)를 아득히 넘으므로, 반드시 **`long`** 타입을 사용해야 한다.

3.  **최고차항 차수**:
    - 수행 횟수가 $n^2$이므로, 시간 복잡도는 $O(n^2)$.
    - 따라서 다항식의 차수는 항상 `2`이다.

## 💻 소스 코드 및 문제 설명
> 백준허브가 자동 생성한 **문제 폴더**로 이동합니다.

- [📂 문제 풀이 폴더 열기 (소스코드 & README)](../../백준/Bronze/24264. 알고리즘 수업 － 알고리즘의 수행 시간 3/)

```java
import java.util.Scanner;

public class Main {

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        // 입력값이 최대 500,000이므로 제곱하면 int 범위를 넘음 -> long 사용 필수
        long n = sc.nextLong();

        // 1. 수행 횟수 출력 (n * n)
        System.out.println(n * n);
        
        // 2. 차수 출력 (이중 반복문이므로 항상 2)
        System.out.println(2);
    }
}
```

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **틀린 이유 (Overflow):** 
	- $n$을 `int`로 선언하여 $n \times n$ 계산 시 약 21억을 초과하는 값이 들어왔을 때 오버플로우가 발생, 음수나 엉뚱한 값이 출력됨.
	- 15894번 문제와 똑같은 실수를 반복함. 입력값 범위를 보고 $21$억이 넘는지 확인하는 습관이 아직 부족함.
- **배운 점:** 
	- 입력값의 범위($500,000$)를 보고 연산 결과($n^2$)가 `int` 범위를 넘는지 미리 계산(추산)해야 한다.
	- $n$이 $5$만 이상이고 제곱 연산이 있다면 `long`을 고려하자. ($46340^2$부터 `int` 범위 초과)
