## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

- **핵심 조건**: 회원 정보(나이, 이름)를 나이 오름차순으로 정렬하되, **같은 나이는 가입 순서(입력 순서)를 유지**해야 한다.

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

1. **변수 설정:** `String[n][2]` 2차원 배열로 나이(`[i][0]`)와 이름(`[i][1]`)을 함께 저장
2. **조건문 설계:** `Arrays.sort()`에 람다 Comparator를 사용, 나이를 `Integer.parseInt()`로 변환해 숫자 비교 → Java의 `Arrays.sort`는 **stable sort**이므로 같은 나이는 입력 순서 보장
3. **시간 복잡도:** O(n log n) — Arrays.sort 기반 Tim Sort

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

- **틀린 이유:**
  1. `bad operand types for binary operator '-'` 
	  - 나이를 String으로 저장한 뒤 `o1[0] - o2[0]` 처럼 뺄셈 연산 시도 → String끼리 `-` 불가
  2. 메모리 초과 
	  - 배열 선언을 `new String[n][n]`으로 해서 n×n 크기 할당 → `new String[n][2]`로 수정
  3. `compareTo()`로 문자열 비교 시 `"9" > "21"` 처럼 사전순 오류 발생 가능

- **배운 점:**
  - String 비교는 `compareTo()`, 숫자 비교는 `Integer.parseInt()` 후 `-` 연산 사용
  - 2D 배열 크기는 실제 필요한 열 수(`[n][2]`)로 정확히 선언할 것
  - Java `Arrays.sort()`는 stable sort(Tim Sort) → 같은 키 값의 원소 순서 보장됨
