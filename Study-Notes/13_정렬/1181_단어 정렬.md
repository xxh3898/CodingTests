# 2026-03-21
## 💡 문제 이해
> 단어 N개를 입력받아 두 가지 기준에 따라 정렬하고, 중복을 제거한 뒤 출력하는 문제입니다.

- **핵심 조건**: 
    - **길이가 짧은 것**부터 정렬
    - **길이가 같으면 사전 순**으로 정렬
    - **중복된 단어**는 하나만 남기고 출력

---

## 🚀 접근 방식 (Logic)
> 정렬 기준이 두 개인 경우 커스텀 정렬(Comparator)을 활용해야 합니다.

1. **변수 설정:** 
    - `String[] arr`: 입력받은 단어들을 저장할 배열
    - `int n`: 입력받을 단어의 개수
2. **조건문 설계 (정렬 로직):** 
    - `Arrays.sort()`와 `Comparator`를 사용하여:
        - 두 단어의 길이가 다르면 `len1 - len2` (오름차순).
        - 길이가 같으면 `word1.compareTo(word2)` (사전 순).
3. **중복 제거 출력:** 
    - 정렬 후, 현재 인덱스의 단어가 이전 인덱스의 단어와 다를 때만 출력합니다.
4. **시간 복잡도:** 
    - $O(N \log N)$

## 📝 오답 노트 & 회고
> 발생했던 문법 에러와 논리적 실수를 기록합니다.

- **틀린 이유:** 
    1. **자바 배열 문법 오류**: 배열의 길이는 메서드인 `.length()`가 아니라 속성인 `.length`를 사용해야 하는데 혼동함 (`Cannot invoke length() on the array type String[]`).
    2. **Arrays.sort() 사용법**: `Arrays.sort()`에는 정렬할 배열 자체를 전달해야 하는데, 배열의 크기나 엉뚱한 값을 전달하려 함.
    3. **문자열 비교 방식**: 문자열의 내용을 비교할 때는 `==`, `!=`가 아닌 `.equals()`를 사용해야 함을 간과함.

- **배운 점:** 
    - 자바 배열과 객체(String)의 메서드/속성 차이를 확실히 알게 됨.
    - 정렬된 데이터에서 `if (!arr[i].equals(arr[i-1]))` 패턴을 사용하면 선형 시간($O(N)$) 안에 중복을 제거하며 출력할 수 있음을 배움.
