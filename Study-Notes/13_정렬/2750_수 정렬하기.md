# 2026-03-10

## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

N개의 정수가 주어졌을 때 **오름차순으로 정렬하여 출력하는 문제**.

- **입력**
	- 첫 줄: 정수 개수 N
	- 다음 N줄: 정수
- **출력**
	- 정렬된 정수를 한 줄씩 출력
- **핵심 조건**
	- 1 ≤ N ≤ 1000
	- 정렬 알고리즘 직접 구현 가능
	- 시간 제한 내에서 **O(N²)** 정렬 사용 가능

---

## 🚀 접근 방식 (Logic)
> 코드를 짜기 전, 어떤 알고리즘과 자료구조를 쓸지 설계하는 단계입니다.

### 1. 변수 설정
- `n` : 입력받을 숫자의 개수
- `arr[]` : 입력된 숫자를 저장할 배열
- `temp` : 두 값을 교환할 때 사용할 임시 변수

입력 과정
- Scanner로 `n` 입력
- 크기 `n` 배열 생성
- 반복문으로 배열에 값 저장

---

### 2. 조건문 설계

사용 알고리즘: **버블 정렬 (Bubble Sort)**

정렬 방식
1. 인접한 두 값을 비교한다.
2. 앞의 값이 더 크면 두 값을 교환한다.
3. 이 과정을 배열 전체에 대해 반복한다.
4. 반복이 진행될수록 큰 값이 배열 뒤쪽으로 이동한다.

구조
```
for (i = 0 → n)
    for (j = 0 → n-1)
        arr[j] > arr[j+1] 이면 swap
```

예시

입력
```
5 4 3 2 1
```

정렬 과정
```
1회 → 4 3 2 1 5
2회 → 3 2 1 4 5
3회 → 2 1 3 4 5
4회 → 1 2 3 4 5
```

---

### 3. 시간 복잡도

버블 정렬 구조
```
외부 반복: N
내부 반복: N
```

총 연산량
```
N × N = N²
```

시간 복잡도
```
O(N²)
```

문제 조건

```
N ≤ 1000
```
최대 연산
```
1000 × 1000 = 1,000,000
```

일반적인 알고리즘 문제 기준 **충분히 제한 시간 내 실행 가능**.

## 📝 오답 노트 & 회고
> 틀렸다면 왜 틀렸는지, 맞았다면 더 좋은 방법은 없는지 기록하세요.

### 틀린 이유

초기 구현에서는 배열을 **한 번만 순회하며 swap**을 수행했다.

```
for (i = 0 → n-1)
```

이 경우 전체 정렬이 완료되지 않는다.

이유
- 버블 정렬은 **반복적으로 배열을 순회해야 정렬이 완성된다**
- 한 번의 순회에서는 **가장 큰 값만 배열 뒤로 이동**

따라서 **이중 반복문 구조로 수정해야 정상 정렬이 된다**.

---

### 배운 점

1. 버블 정렬은 반드시 **이중 반복문 구조**를 사용한다.
```
for (전체 반복)
    for (인접 비교)
```

2. 정렬 알고리즘은 **단순 swap 구현이 아니라 동작 원리 이해가 중요하다**.

핵심 개념
- 한 번 반복 → 가장 큰 값이 뒤로 이동
- 반복 누적 → 전체 정렬 완료

---

### 다른 사람 풀이 참고

Java에서는 기본 정렬 메서드를 사용할 수도 있다.

```
Arrays.sort(arr);
```

시간 복잡도

```
O(N log N)
```

하지만 이 문제의 목적은 **기본 정렬 알고리즘 구현 연습**이므로  
버블 정렬 / 선택 정렬 / 삽입 정렬을 직접 구현하는 풀이가 일반적이다.
