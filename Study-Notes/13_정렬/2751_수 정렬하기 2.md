# 2026-03-16

## 💡 문제 이해
> 문제를 나만의 언어로 요약해서 적어보세요.

정수 N개가 주어지면 이를 오름차순으로 정렬해서 출력하는 문제.

- **핵심 조건**
	- N ≤ 1,000,000
	 - 시간 제한: 2초
	- 많은 입력 데이터를 처리해야 하므로 빠른 입출력이 필요

---

## 🚀 접근 방식 (Logic)

1. **변수 설정**
	- `int n` : 입력 받을 숫자 개수
	- `long[] arr` : 정렬할 숫자 배열

2. **조건문 설계**
	- 반복문으로 N개의 숫자를 배열에 저장
	 - `Arrays.sort()`를 사용하여 오름차순 정렬
	 - 정렬된 배열을 순서대로 출력

3. **시간 복잡도**
	- `Arrays.sort()` 시간복잡도  
	  → **O(N log N)**

계산 과정
N = 1,000,000일 때
log₂(1,000,000) ≈ 20
따라서
```
N log N
= 1,000,000 × 20
= 약 20,000,000 연산
```
이는 2초 제한에서 충분히 처리 가능하다.
하지만 입력이 매우 많기 때문에 **Scanner 대신 BufferedReader 사용이 필요하다.**

## 📝 오답 노트 & 회고

### 틀린 이유
- `Scanner`와 `System.out.println()`을 사용하여 대량의 입력과 출력을 처리하면서 **입출력 속도가 느려져 시간 초과가 발생했다.**
  Scanner는 내부적으로 정규식 기반 파싱을 수행하여 BufferedReader보다 입력 속도가 느리다.
  또한 `System.out.println()`을 반복 호출하면 출력 버퍼를 효율적으로 사용하지 못해 성능이 떨어진다.

### 배운 점
- 입력 데이터가 많은 문제에서는 **BufferedReader + BufferedWriter**를 사용해야 한다.
- 알고리즘 시간복잡도뿐 아니라 **입출력 방식도 시간초과의 원인이 될 수 있다.**

### 다른 사람 풀이 참고
대부분의 풀이가 다음 구조를 사용한다.

```
BufferedReader
BufferedWriter
Arrays.sort()
```

대량 데이터 문제에서는 Java에서 **빠른 입출력 패턴이 사실상 필수**라는 것을 확인했다.
